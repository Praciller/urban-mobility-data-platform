# Portfolio Review

## Phase 4 Review Path

1. Inspect `PROJECT_REQUIREMENTS_NO_SUPABASE.md` and `docs/architecture.md`.
2. Review the external `DATA_DIR` configuration in `.env.example`.
3. Run the bounded validation, DuckDB load, and dbt pipeline.
4. Start FastAPI and inspect `/health`, `/metadata`, `/docs`, and filtered metrics.
5. Run lint, format checks, tests, pre-commit, and repository guardrails.
6. Review read-only repository SQL, response schemas, and generated-fixture API tests.

The complete product review path will expand when the dashboard and Dagster phases ship.
